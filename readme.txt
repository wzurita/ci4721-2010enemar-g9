convenciones que debemos utilizar:
  para los nombres de archivos y carpetas solo estan permitidos:
    letras en minusculas {'a',...,'z'}
    numeros {'0',...,'9'}
    subguion {'_'}
  la barra a usar para las direcciones es '/'
  al realizar un cambio, no detenerse hasta hacer commit.
  acompañar cada commit con su mensaje respectivo explicando de manera resumida los cambios realizados
  hacer commit de versiones funcionales (evitar subir errores)
  en caso de subir un error, correguirlo a la brevedad posible

  Deberiamos definir cuales son las reglas del editor de texto:
    yo uso identacion de 2 espacios y reemplazo los tabs con espacios
  
a continuacion desbribo el significado de cada carpeta
  teoria/jesus/...
    contine mis archivos y notas de las clases... solo en caso que sean necesarias.
    en esta carpeta las convenciones no aplican.
  teoria/gustavo/...
    el espacio designado para que coloques tus notas si lo deseas
    en esta carpeta las convenciones no aplican.
  laboratorio/entregaX/... (con X perteneciente a {1,2,3,4}
    es el espacio para conservar las 4 entregas del curso sin modificaciones... no sabemos si las necesitaremos a futuro
    es posible que sea una copia exacta de laboratorio/actual/... en el momento de la entrega
  laboratorio/actual/java/...
    contiene el codigo java de nuestro proyecto.
  laboratorio/actual/doc/mips/...
    contiene la documentacion de mips que pueda ser util para nuestro proyecto.
  laboratorio/actual/doc/java/...
    contiene la documentacion de java que pueda ser util para nuestro proyecto.
  laboratorio/actual/doc/???/...
    contiene la documentacion de nuestro proyecto. (stuff burocratica)
  laboratorio/actual/tests/...
    contiene archivos con programas para probar la funcionalidad (en distintos niveles de dificultad) de nuestro lenguaje
    cada archivo de prueba que usemos para probar el lenguaje deberia ser guardado en esta carpeta
    por si queremos repetir las pruebas luego de algun cambio.
